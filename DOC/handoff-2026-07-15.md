## สถานะล่าสุด

งานรอบนี้ปรับหน้าจอ authentication ให้เป็น UI แบบ two-column พร้อมภาพ globe และปรับ form ให้ใช้งานสะดวกขึ้น โดยโค้ดทั้งหมดอยู่ใน working tree และยังไม่ได้ commit ณ เวลาที่เขียนเอกสารนี้

## งานที่ทำแล้ว

- เพิ่ม `AuthShell` และ `AuthGlobe` เพื่อเป็น shared layout ของหน้า `/login` และ `/register` พร้อมพื้นหลัง globe แบบ interactive
- ปรับหน้า Login: โลโก้/หัวข้อใหม่, email และ password input พร้อมไอคอน, ปุ่ม GitHub/Google และ divider
- ปรับหน้า Register: name/email, confirm password, ตรวจรหัสผ่านตรงกัน และ password-strength indicator
- ปรับ `PasswordInput`: icon lock, preserve `className`, และปุ่ม text สำหรับ show/hide password
- ปรับ theme toggle ให้จำค่าใน `localStorage` และเพิ่ม initial-theme script ใน root layout เพื่อลดอาการ theme กระพริบตอนโหลดหน้า
- ปรับ header ให้ sticky และมีพื้นหลังขณะเลื่อนหน้า
- ปรับ `Separator` ให้ใช้ border colour ที่กลมกลืนขึ้น
- เพิ่ม skill ภายในโปรเจกต์ `.agents/skills/react-icons-ci` สำหรับบังคับมาตรฐานไอคอน Circum Icons

## จุดที่ควรทำต่อ / ตรวจสอบก่อนปล่อยจริง

1. รัน `npm.cmd run build` และทดสอบหน้า `/login` กับ `/register` บน browser จริง
2. Login และ Register ยังเรียก API เดิม; ต้องทดสอบ successful/error response รวมถึง redirect ไป `/dashboard`
3. ลิงก์ “Forgot password?” ยังชี้กลับไป `/login` เพราะยังไม่มี flow reset password
4. OAuth buttons ใช้ `FaGithub` และ `FcGoogle`; ขัดกับ skill ไอคอนของโปรเจกต์ที่ให้ใช้เฉพาะ `react-icons/ci` (หรือควรเปลี่ยนเป็น text link หากไม่มี Circum brand icon)
5. ตรวจ performance ของ WebGL globe บนมือถือ/เครื่องสเปกต่ำ และเตรียม fallback หากจำเป็น
